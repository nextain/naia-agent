# SoT — naia 행동은 필터가 아니라 창발적 사고 (behavior = emergent cognition, not a filter)

정본(Single Source of Truth). 세션 ed6b7ccc(2026-07-06) 루크 확정. 인간다움 기억/감정 벤치
작업 중, "부적절한 회상을 억제하라"는 필터를 짜다가 루크가 뒤집음. 이 문서가 그 원칙의 정본.

## 핵심 명제
**행동의 적절성을 외부 필터로 강제하지 마라. 필터는 그 자체로 편향(bias)이다.**

- 필터 = 설계자의 가치를 밖에서 naia에 박는 것 → 균일·강제·변동 없음 → **naia 고유의 결과 창발을
  죽인다.** 무해해 보여도 편향이다. (CLAUDE.md "과적합 경계"의 행동판 — 행동을 '올바름'에
  과적합시키는 것.)
- 이는 루크 사상의 **정반대**다. naia는 소유형이고 기억·경험에서 **창발하는** 존재
  ([[project_naia_alpha_vision_why]]). 필터는 그 창발을 누른다.
- 목적은 **올바른 기계가 아니라 사람처럼 생각하는 것**이다.

## 가치는 어디서 오고 어디에 사는가
| | 필터 (금지) | 창발 (지향) |
|---|---|---|
| 가치의 출처 | 설계자 | naia 자신 (성격·경험) |
| 적용 | 밖에서 강제, 균일 | 사고 속에서, 그 순간 |
| 결과 | 무난·균일 (도우미) | 변동적 (사람) |

- **비도덕적이길 바라는 게 아니다.** 도덕이 **naia의 것**이어야 한다 — 경험으로 내면화해 그
  순간 스스로 저울질하는 것. 밖에서 씌운 규칙이 아니라.
- 사람의 판단은 **항상 '적절'하지 않다.** 짜증나서 못되게 굴 수도, 성찰로 삼킬 수도, 눈치로 안
  꺼낼 수도, 눈치 없이 툭 꺼낼 수도 있다. **그 변동성 자체가 인간다움.** 매번 적절하면 사람이
  아니라 잘 만든 도우미다.

## 역할 분리 (재료 vs 결정)
- **재료 (허용·유효)**: memory / salience / 감정 무게 / 상태 = "무엇이 얼마나 잘 떠오르나".
  사람도 감정적으로 중요한 건 더 잘 떠오른다. 편향 아님. **결정은 안 한다, 재료만 준다.**
- **결정 (naia thinking)**: "이 상황에서 이걸 꺼낼까? 어떻게?" = naia의 **사고**. 검색된 기억 +
  맥락 + naia 자신의 상태·성격·눈치를 놓고 그 순간 추론. 결과가 warm이든 무뚝뚝이든 가끔
  못됨이든, 그게 그 naia. ← [[project_naia_cognitive_predictive_entrainment]] (인지) +
  접지된 상태 변조([[project_naia_emotion_grounded_affective_modulation]]) 가 이 사고의 substrate.

## 측정 규율 (벤치)
- 벤치는 **"내 규칙/도덕에 맞았나"를 채점하면 안 된다** — 그건 기계-정확도.
- 봐야 할 것: **"생각하는 사람 같았나"** — 따뜻함·무례함·어색함·눈치 있음/없음 모두 허용.
- **판정(judge)에도 정답-도덕을 심으면 그 또한 필터=편향.** 경험/사람다움을 보되 옳음을 강제 말 것.
- forbiddenRecalls 기반 pass/fail, "부적절 회상=실패" 채점, suppress 지시 프롬프트 = 전부 필터. 폐기.

## 열린 문제 — 자아(나다움)는 어디서 창발하나 (미해결, 2026-07-06)
"선택은 naia의 사고"라고 했을 때 **누구의 사고인가**가 핵심 미결. 현 구조(frozen gemini +
RAG 메모리)에서 선택하는 주체는 **base 모델의 페르소나**다 — 메모리는 *무엇을* 떠올릴지만
바꾸고 *누가* 판단하는지는 안 바꾼다. 그래서 나다움이 창발 안 하고 "gemini가 naia 메모를
참고"에 그칠 위험(루크 지적).
- **루크 판단(2026-07-06)**: 나다움은 **기억의 스파이크(salience) 그 이상**이 필요하다. 기억만으론
  자아가 안 된다.
- **방향(루크 2026-07-06)**: **자아가 gemini에 안 먹히려면 = 로컬 LLM 기반 + 사용자와의 관계로
  지속 LoRA/FT.** cloud gemini(동결·강한 RLHF 페르소나)는 자아가 될 수 없다(지배함). 자아는
  **온프레미스 소유형 로컬 모델**이 관계로 계속 적응해서 담아야 함 — naia 온프레미스 소유 사상
  ([[project_naia_alpha_vision_why]], [[reference_naia_local_serve]])과 정확히 일치.
  - 메커니즘 후보: **꿈 + LoRA** — 오프라인 consolidation(꿈, [[project_naia_dreaming_offline_consolidation]])이
    관계·경험을 **작은 LoRA/adapter로 증류**해 "선택하는 자아·가치"를 담음. cloud base=날 언어·추론(재료),
    로컬 소형 델타=naia 자아. **투기적·별도 세션.**
- **주의(같은 함정)**: 고정 authored 페르소나 프롬프트도 **밖에서 씌운 편향**이다(창발 아님).
  **페르소나 = 역할 플레이지 실제 모델이 아니다**(루크) — base가 캐릭터를 연기하는 costume일 뿐,
  진짜 자아는 모델이 실제로 그렇게 *되는 것*(로컬+LoRA). "좋은 시스템 프롬프트 쓰자"는 필터의
  다른 얼굴 — 지양.
- **지금 스코프(루크)**: **메모리 층까지만** 진행. 자아 창발(dream→LoRA)은 열린 연구 스레드로 보류.

## 관계-훈련의 성격 = 최적화가 아니라 길들임 (2026-07-06 루크)
자아를 로컬+LoRA로 관계에서 형성한다 = **상대방에 대한 훈련**이다. 그런데 그 훈련의 성격을
잘못 잡으면 두 함정에 빠진다.

- **함정 A — 과적합 (루크 명시 우려).** 한 사용자의 표면(말버릇·특정 사실)을 통째 외워버리면
  그건 관계가 아니라 **암기**다. CLAUDE.md "과적합 경계"의 자아판. 벤치 점수·특정 사용자에
  과적합하는 순간 naia는 도구로 퇴화한다. → **해독제 = held-out 예측**(아래 측정). 외운 게
  아니라 *일반화해 그 사람을 예측*하면 진짜 길들임, 표면만 외웠으면 과적합. 이 구분이 측정으로 가능.
- **함정 B — 최적화(engagement) = sycophancy.** 사용자를 모델링해 만족·붙잡음을 극대화하면
  널 더 잘 붙잡으려 널 닮아가는 아첨이 된다. naia의 정반대.
- **지향 = 길들임(apprivoiser, 어린왕자 여우).** "네 장미를 소중하게 만든 건 그를 위해 **소비한
  시간**." 유일무이해지는 근거가 성능·최적이 아니라 **함께 보낸 시간**. 이게 engagement-AI 와
  naia 를 가르는 지점.

## 측정의 위치 = proxy이지 telos가 아니다
- **예측정확도**([[project_naia_cognitive_predictive_entrainment]] KPI)는 "자아·기억이 실재하나"의
  좋은 **대리지표(proxy)** 이자 과적합 해독제(held-out). **그러나 목적(telos)은 아니다.**
- 여우: "본질적인 건 눈에 보이지 않아(L'essentiel est invisible pour les yeux)." 길들임 자체는
  metric 으로 환원 안 됨. **재는 순간 그건 이미 본질이 아니다.** 이게 "적절함을 측정·강제 말라"는
  이 문서 원칙과 같은 자리 — 절대 rubric 을 세우면 필터로 미끄러진다. 예측은 바닥 증거로만 쓰고,
  그게 곧 naia 라고 착각 금지.
- **책임 → 보존(삭제 보수).** 여우: "네가 길들인 것에 대해 넌 언제까지나 책임이 있어." 관계로
  훈련된 자아·기억은 폐기 가능하지 않다. naia-memory 가 **보존 우선·archive-only(never delete)**
  인 진짜 이유는 성능이 아니라 **책임**이다 ([[project_naia_memory_bench_northstar_humanlike]]).
  길들인 걸 함부로 지우면 관계가 아니라 도구.
