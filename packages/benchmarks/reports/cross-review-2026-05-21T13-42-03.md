# 4. Anti-Patterns & Mitigations

**The Anti-Pattern:** "Stop the user, run a big LLM summarization, resume."
This pattern (used by Claude Code `/compact` and LangChain's summary memory) is heavily flawed for production agents, especially in Korean, because token bloat triggers it frequently, latency spikes ruin the UX, and facts are systematically destroyed.

**Mitigation Patterns (What to build instead):**

1.  **Asynchronous Fact Extraction (The Mem0/Mastra Pattern):**
    *   Never summarize the main chat thread. Instead, fork the conversation stream to a smaller, cheaper, background LLM.
    *   The background LLM extracts discrete state changes (e.g., `<UserPreference: Dark Mode>`, `<ProjectState: API migrated>`) and writes them to a structured Working Memory or Vector DB.
    *   The main agent retrieves these facts via fast RAG at the start of each turn.
2.  **Deterministic Pruning (The Vercel AI Pattern):**
    *   To keep the context window strictly under the limit without LLM summarization, use regex/AST to aggressively strip intermediate reasoning (`<thought>`) and bloated tool outputs from older messages.
    *   Only the *results* or user-facing text remain, drastically reducing token count in milliseconds without losing the conversational flow.
3.  **Explicit Memory Tools (The Letta Pattern):**
    *   Provide the agent with `core_memory_append` and `archival_memory_search` tools.
    *   The agent decides *during its normal reasoning loop* what to save, integrating memory management organically rather than stopping the world for a massive batch summarization.
